### Solo Mode:
- MLFlow: The main aim is get the best model, which is achieved by comparision. It is used to track(logginng & quering) experiments, runs and artificats for different models. It can store parameters, metrics, artifacts (models, data).

- MLflow Tracking: A server where experiments are logged. The local logs genereated by MLFlow can be managed from any where.


- 
```sh
pip install mlflow
mlflow ui
```
- Now we can get `http://127.0.0.1:5000` the experiments are stored in **default** component.
